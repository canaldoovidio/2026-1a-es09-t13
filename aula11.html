PARTE 1

ESCALA

O Problema da Escala

description

50+ Features

Organizadas por domínio, não por sprint. Domínios são estáveis; sprints mudam a cada duas semanas.

checklist

200+ Cenários

Tags para filtrar, priorizar e executar. Sem tags, você roda tudo ou nada.

hub

15+ Integrações

Cada uma com tática, mock e monitoramento. Integração sem tática é ponto de falha.

Requisitos como código também precisam de arquitetura. Sem organização, 200 cenários viram 200 problemas.

ISO 25010

ISO 25010: RNF com Vocabulário Padronizado

Característica Sub-característica Métrica Teste
Performance Tempo de Resposta p95 < 200ms Locust
Confiabilidade Tolerância a Falhas MTTR < 5min CB test
Segurança Autenticidade 0 acessos não autorizados BDD segurança
Manutenibilidade Modificabilidade Cobertura > 80% pytest-cov

ISO 25010 fornece vocabulário padronizado — use nos seus RNFs. Sub-características da norma evitam ambiguidade.

EVOLUÇÃO

Do Mock à Integração Real

science

Nível 1: Tudo Mockado

Dev local. MagicMock para tudo. Rápido e isolado. Ideal para desenvolvimento inicial.

cloud_sync

Nível 2: Mocks + Staging

Mocks para instáveis, real em staging. Mais confiável. Detecta problemas de contrato.

verified

Nível 3: Ambiente Real

Testes contra prod com feature flags. Máxima confiança. Requer infra madura.

Vocês estão no Nível 1. O objetivo é chegar ao Nível 2. Mocks realistas com side_effect são o caminho.

MÉTRICAS

Métricas de Processo

fact_check

Cobertura de RF

% de regras de negócio com cenário BDD. Meta: 100% dos RFs críticos cobertos.

verified

Cobertura de RNF

% de atributos de qualidade com teste automatizado. Inclui performance, resiliência e segurança.

trending_down

Taxa de Regressão

Cenários que quebraram no último sprint. Alta taxa = arquitetura frágil ou testes acoplados.

assignment_late

Cenários planejados mas não implementados. Dívida de teste cresce silenciosamente.

LIÇÕES

Lições Aprendidas

folder

Organize por domínio, não por sprint

Sprint muda a cada duas semanas. Domínio permanece. A estrutura de pastas deve refletir a arquitetura, não o cronograma.

swap_horiz

Rastreabilidade bidirecional = auditabilidade

Do requisito ao teste e vice-versa. Se você não consegue navegar nos dois sentidos, não é rastreável.

menu_book

ISO 25010 = vocabulário padronizado

Use sub-características da norma nos seus RNFs. Evita ambiguidade e facilita comunicação com stakeholders.

security

O pipeline é o guardião

Requisito que não é gate no CI/CD não é enforced. Se não bloqueia o deploy, é só documentação.
