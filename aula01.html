ENTERPRISE VIEWPOINT

Visão de Negócio

Foco

Propósito, escopo e políticas do sistema. Descreve os requisitos de negócio e como atendê-los.

Perguntas-chave

  • Por que o sistema existe?
  • Quais são os objetivos de negócio?
  • Quem são os stakeholders?
  • Quais políticas governam o sistema?

Exemplo ASIS

Objetivo: Construir um sistema digital resiliente com qualidade como ativo de software

Stakeholders: Clientes (empresas), Fisco, Equipe de desenvolvimento

Política: Compliance tributário 100%, disponibilidade 99.9%

INFORMATION VIEWPOINT

Visão de Informação

Foco

Semântica da informação e processamento de dados. Descreve a informação gerenciada pelo sistema.

Perguntas-chave

  • Quais dados o sistema manipula?
  • Qual a estrutura e tipos dos dados?
  • Como os dados se relacionam?
  • Quais são as regras de integridade?

Exemplo ASIS

Dados: Notas fiscais, cálculos tributários, cadastros de empresas

Estrutura: Schemas XML/JSON conforme padrões SPED

Integridade: Validação de CNPJ, consistência de valores

COMPUTATIONAL VIEWPOINT

Visão Computacional

Foco

Decomposição funcional do sistema em objetos que interagem via interfaces. Descreve a funcionalidade fornecida.

Perguntas-chave

  • Quais são os componentes funcionais?
  • Como os componentes interagem?
  • Quais são as interfaces (APIs)?
  • Quais operações cada componente oferece?

Exemplo ASIS

Componentes: Serviço de Cálculo, Serviço de Validação, Gateway de Integração

Interfaces: REST APIs documentadas (OpenAPI)

Operações: calcularImposto(), validarNota(), enviarSPED()

ENGINEERING VIEWPOINT

Visão de Engenharia

Foco

Mecanismos e funções para suportar interações distribuídas. Descreve a distribuição do processamento.

Perguntas-chave

  • Como os componentes são distribuídos?
  • Quais mecanismos de comunicação?
  • Como garantir transparência de distribuição?
  • Qual a estratégia de escalabilidade?

Exemplo ASIS

Distribuição: Microserviços em containers Kubernetes

Comunicação: REST síncrono + Filas assíncronas

Escalabilidade: Auto-scaling horizontal por demanda

TECHNOLOGY VIEWPOINT

Visão Tecnológica

Foco

Escolha de tecnologias para implementar o sistema. Descreve as tecnologias escolhidas para processamento e apresentação.

Perguntas-chave

  • Quais linguagens e frameworks?
  • Quais bancos de dados?
  • Qual infraestrutura de cloud?
  • Quais ferramentas de DevOps?

Exemplo ASIS

Backend: Java/Spring Boot, Node.js

Dados: PostgreSQL, Redis, Elasticsearch

Infra: AWS/Azure, Docker, Kubernetes

Cenários de Arquitetura

O que são Cenários de Arquitetura?

São descrições de situações específicas que o sistema deve suportar, expressas como testes automatizados que validam atributos de qualidade.

Estímulo

O que dispara o cenário?

Ex: 1000 requisições simultâneas
→

Ambiente

Em qual contexto?

Ex: Horário de pico, sistema em produção
→

Resposta

O que deve acontecer?

Ex: Latência < 200ms, 0 erros

Exemplo: Cenário de Performance

Python / pytest test_performance.py 
import pytest
from locust import HttpUser, task, between

@pytest.mark.performance
class TestCalculoTributario:
    """
    Cenário: Cálculo tributário sob carga
    Business Driver: Suportar 10.000 req/min
    """
    
    def test_latencia_calculo(self, api_client):
        # Estímulo: 1000 requisições simultâneas
        response = api_client.post("/api/v1/calculo", 
            json={"cnpj": "12345678000199"})
        
        # Resposta esperada: latência < 200ms
        assert response.elapsed.total_seconds() < 0.2
        assert response.status_code == 200
link

Conexão: Este teste valida o Business Driver de volumetria (10.000 req/min) através da visão de Engineering.

Exemplo: Cenário de Segurança

Python / pytest test_security.py 
@pytest.mark.security
class TestAutenticacaoAPI:
    """
    Cenário: Proteção de endpoints sensíveis
    Business Driver: Compliance LGPD
    """
    
    def test_acesso_sem_token(self, api_client):
        # Estímulo: Requisição sem autenticação
        response = api_client.get("/api/v1/dados-fiscais")
        
        # Resposta: Acesso negado (401)
        assert response.status_code == 401
    
    def test_token_expirado(self, api_client, expired_token):
        # Estímulo: Token JWT expirado
        headers = {"Authorization": f"Bearer {expired_token}"}
        response = api_client.get("/api/v1/dados-fiscais", 
            headers=headers)
        
        assert response.status_code == 401
