CASO 1

SEFAZ — Retry + Circuit Breaker

error_outline Problema

SEFAZ tem janelas de manutenção não anunciadas. O sistema precisa continuar operando mesmo sem emissão de NF-e.

check_circle Solução

CB com timeout 3s + retry exponencial + fallback para fila de contingência.

@circuit_breaker(failure_threshold=3, reset_timeout=30)
def emitir_nfe(dados):
    try:
        response = sefaz_client.post("/nfe", json=dados, timeout=3)
        return response.json()
    except Timeout:
        fila_contingencia.enqueue(dados)
        return {"status": "contingencia", "msg": "NF-e em fila"}

O CB protege O SISTEMA, não o serviço externo.

CASO 1

Testando SEFAZ com BDD

Feature (Gherkin)

@rnf @resiliencia @falha
Feature: Resiliência na emissão de NF-e

  Scenario: CB abre com SEFAZ em manutenção
    Given a SEFAZ está em manutenção
    And o circuit breaker está CLOSED
    When 3 tentativas de emissão falham
    Then o CB muda para OPEN
    And NF-e é enviada para fila de contingência

Step Definitions

@given('a SEFAZ está em manutenção')
def step_sefaz_manutencao(context):
    context.sefaz = MagicMock()
    context.sefaz.post.side_effect = TimeoutError(
        "SEFAZ indisponível - janela de manutenção"
    )

@then('NF-e é enviada para fila de contingência')
def step_fila_contingencia(context):
    context.fila.enqueue.assert_called_once()
CASO 2

BACEN/PIX — Timeout + Bulkhead

error_outline Problema

PIX deve confirmar em < 10s por regulação do BACEN. Falhar não é opção.

check_circle Solução

Timeout estrito de 8s + bulkhead isolando thread pool de pagamentos.

# Bulkhead: pool isolado para PIX
pix_executor = ThreadPoolExecutor(max_workers=10, thread_name_prefix="pix")

async def confirmar_pix(transacao_id):
    future = pix_executor.submit(bacen_client.confirmar, transacao_id)
    try:
        return future.result(timeout=8)  # Timeout regulatório
    except TimeoutError:
        return {"status": "timeout", "acao": "reverter"}

RNFs regulatórios não são negociáveis.

CASO 2

Testando Timeout do PIX

Feature (Gherkin)

@rnf @performance @falha
Feature: Timeout regulatório do PIX

  Scenario: PIX excede timeout de 8s
    Given o BACEN está com latência alta
    When solicito confirmação de PIX
    And a resposta demora mais de 8 segundos
    Then a transação é marcada como timeout
    And uma reversão automática é iniciada

Step Definitions

@given('o BACEN está com latência alta')
def step_bacen_lento(context):
    context.bacen = MagicMock()
    def resposta_lenta(*args):
        time.sleep(10)  # Simula 10s > timeout de 8s
        return {'status': 'aprovado'}
    context.bacen.confirmar.side_effect = resposta_lenta

@then('a transação é marcada como timeout')
def step_timeout(context):
    assert context.resultado['status'] == 'timeout'
CASO 3

Transportadora — Polling + Cache

error_outline Problema

API de rastreio com rate limit de 100 req/min. Consultas excessivas resultam em bloqueio.

check_circle Solução

Polling a cada 5 min + cache local com TTL de 5 min. Respeita o rate limit sem perder dados.

dangerous

Sem Cache

Cada requisição do usuário = chamada à API externa. 500 usuários = 500 req/min. Bloqueio em 12s.

cached

Com Cache

1 req/5 min por pacote. 500 usuários = mesma resposta do cache. ~20 req/min.

Nem toda integração é síncrona. Adapte a tática ao perfil.

SÍNTESE

Lições do Caso

lightbulb

Cada integração tem perfil diferente de falha

SEFAZ cai sem aviso, BACEN tem SLA regulatório, Transportadora tem rate limit, ERP precisa de consistência.

lightbulb

A tática depende do SLA do serviço externo

Não existe tática universal. O perfil de falha do serviço externo define qual mecanismo usar.

lightbulb

Mocks devem simular comportamento REAL

Incluindo latência, timeouts e side effects. Um MagicMock() genérico não testa resiliência.

lightbulb

Monitore ANTES de precisar debugar

Métricas, logs e alertas devem existir antes do primeiro incidente, não depois.

TROUBLESHOOTING

Problemas Frequentes nos Projetos

bug_report

Mock sem side_effect

MagicMock() genérico não testa resiliência. Use side_effect=TimeoutError para simular falhas reais.

label_off

Feature sem tags

Sem @rf/@rnf/@sucesso/@falha = impossível filtrar e executar seletivamente. Tags são obrigatórias.

sync_problem

CB sem HALF-OPEN

Circuit Breaker que nunca tenta recuperar. Sem HALF-OPEN, o sistema fica permanentemente degradado.

speed

Carga em 1 endpoint

Locust testando só GET /health não revela gargalos reais. Distribua a carga pelos endpoints críticos.
